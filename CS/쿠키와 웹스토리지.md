## cookie, sessionStorage, localStorage 사이의 차이점을 설명하세요.

### 쿠키 (Cookie)

- 정의: 웹 서버가 사용자의 브라우저에 보내는 작은 크기의 데이터 조각입니다. 브라우저는 이 데이터를 저장하고, 다음에 같은 서버의 웹 페이지를 요청할 때 다시 서버로 보냅니다.
- 주요 용도:
  - 사용자 인증 (로그인 상태 유지)
  - 사용자 설정 저장 (테마, 언어 등)
  - 사이트 방문 기록 추적
- 특징:
  - 크기 제한: 작은 크기의 데이터만 저장 가능
  - 수명: 만료 기간 설정 가능 (영구 쿠키, 세션 쿠키)
  - HTTP 요청마다 전송: 서버와 클라이언트 간의 모든 HTTP 요청에 포함되어 전송
  - 도메인별 저장: 각 도메인마다 별도의 쿠키 저장
- 단점:
  - 보안: 쿠키는 쉽게 조작될 수 있으므로 민감한 정보를 저장하기에는 부적절
  - 성능: 모든 HTTP 요청에 포함되어 네트워크 트래픽 증가
  - 용량 제한: 저장 가능한 데이터 크기가 제한적

### 웹 스토리지 (Web Storage)

- 정의: HTML5에서 도입된 브라우저 저장소로, 쿠키보다 더 많은 양의 데이터를 저장할 수 있습니다. 로컬 스토리지와 세션 스토리지 두 가지 종류가 있습니다.
- 주요 용도:
  - 웹 애플리케이션 데이터 저장 (설정, 데이터 캐싱 등)
  - 오프라인 웹 애플리케이션 구축
- 특징:
  - 크기: 쿠키보다 훨씬 많은 데이터 저장 가능
  - 종류:
    - 로컬 스토리지: 브라우저를 닫아도 데이터가 유지
    - 세션 스토리지: 브라우저를 닫으면 데이터 삭제
  - HTTP 요청: HTTP 요청에 포함되지 않음
  - JavaScript 접근: JavaScript를 통해 쉽게 데이터를 저장하고 읽기 가능
- 장점:
  - 대용량: 많은 양의 데이터 저장 가능
  - 성능: HTTP 요청에 포함되지 않아 네트워크 트래픽 감소
  - 보안: 쿠키보다 상대적으로 안전
- 단점:
  - 호환성: 모든 브라우저에서 완벽하게 지원하지 않을 수 있음

|                                   |                             cookie                              | localStorage | sessionStorage |
| :-------------------------------: | :-------------------------------------------------------------: | :----------: | :------------: |
|              생성자               | 클라이언트나 서버. 서버는 Set-Cookie 헤더를 사용할 수 있습니다. |  클라이언트  |   클라이언트   |
|               만료                |                          수동으로 설정                          |    영구적    |  탭을 닫을 때  |
|    브라우저 세션 전체에서 지속    |                   만료 설정 여부에 따라 다름                    |      O       |       X        |
| 모든 HTTP 요청과 함께 서버로 보냄 |              쿠키는 Cookie 헤더를 통해 자동 전송됨              |      X       |       X        |
|          용량 (도메인당)          |                               4kb                               |     5MB      |      5MB       |
|              접근성               |                           모든 윈도우                           | 모든 윈도우  |    같은 탭     |
